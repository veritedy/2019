HTML

HTML

HTML(HyperText Markup Language)은 웹 페이지를 만들기 위한 언어입니다.


태그(Tag)

태그는 정보를 정의하는 방식을 의미합니다.

  1. 태그는 열린 태그와 닫는 태그가 있습니다.
  2. 닫는 태그에는 '/'가 있어야 합니다.
  3. 닫는 태그가 필요 없는 태그도 있습니다.(<br />, <hr />, <img />, <meta />)
  4. HTML5에서는 '/'가 생략이 가능합니다.

블록 요소/인라인 요소

블록 요소는 박스, 인라인 요소는 텍스트를 의미합니다.

  • 블록 엘리먼트(Block Element)
    • 독립된 박스 영역으로, 한 줄에 하나의 블록 요소만 표현 할 수 있습니다.
    • 블록 요소에는 블록 요소와 인라인 요소를 포함 할 수 있습니다.
    • <div>, <p>, <h1~h6>, <ul>, <li>
  • 인라인 엘리먼트(Block Element)
    • 독립된 텍스트 영역으로, 한 줄에 여러 개의 인라인 요소를 표현 할 수 있습니다.
    • 인라인 요소에는 블록 요소가 포함될 수 없지만, 인라인 요소는 포함 할 수 있습니다.
    • <span>, <a>, <strong>, <em>, <img>

파비콘

인터넷 웹 브라우저의 주소창에 표시되는 웹페이지를 대표하는 아이콘입니다.

  • 웹 브라우저 아이콘입니다.
  • .ico 파일로 만들어야 합니다.
  • .ico 파일은 모든 브라우저를 지원하고 .png 파일은 ie6~10에서는 지원 안됨
  • 모바일 아이콘도 따로 설정 할 수 있습니다.
  • 모바일 아이콘은 해상도별로 설정합니다.

DTD (Document Type Declaration) 선언

웹 페이지를 제작하기 위해서는 웹 문서 형식 유형을 설정하여야 합니다.

독타입을 설정하지 않거나 잘못 설정하면 브라우저에 따라 화면이 다르게 나오거나 렌더링이 달라지기 때문에 독타입을 설정해야 합니다.


언어 속성 설정

컨텐츠의 언어를 인식하고 얻은 정보를 추출하는데 사용합니다.

<!-- hrml5 -->
<html lang="ko">

<!-- hrml4 -->
<html lang="ko" xml;:ang="ko" xmlns="www.w3.org/1999/xhtml"-->

HTML Reference

<div>

블록요소로 웹페이지의 (전체적인 틀)레이아웃을 만들 때 주로 사용합니다.


<a>

연결할 주소를 지정합니다.

  • <a> href: 연결할 주소를 지정하는 속성입니다.
  • <a> title: 해당 링크에 마우스를 갖다대면 링크에 대한 설명이 나옵니다.
  • <a> target: 링크를 클릭할 때 창을 어떻게 열지 결정합니다.

    • <b>

    텍스트를 중요성과 관련성을 전달하지 않고 구별할 때 글자를 기울여서 표시하는 태그입니다.


    <em>

    특정부분을 강조하고 싶을 때 사용하는 태그입니다.


    <i>

    텍스트를 평범한 글자와 구분하기 위해 기울여서 표시하는 태그입니다.


    <mark>

    형광펜으로 그어 놓은 듯한 효과를 내는 태그입니다.


    <span>

    인라인요소로 웹페이지의 일부분에 스타일을 적용시키기 위해 사용합니다. div와 비슷한 기능으로 차이점은 div는 줄 바꿈이 되지만, span은 줄 바꿈이 되지 않습니다.


    <strong>

    중요한 문구를 강조하는 HTML 태그입니다.


    <pre>

    Preformatted Text의 약자로 정해진 서식을 브라우저가 무시하지 않고 공간, 간격, 라인, 띄어쓰기 등을 모두 있는 그대로 나타나도록 정의합니다.


    <table>

    HTML 문서에서 표를 만드는 태그입니다.

  • <table> : 테이블을 만드는 태그입니다.
  • <th> : 테이블의 헤더부분을 만드는 태그입니다.
  • <tr> : 테이블의 행을 만드는 태그입니다.
  • <td> : 테이블의 열을 만드는 태그입니다.

    • <h1~h6>

    h1~h6 요소는 6단계의 문서 제목을 구현합니다. 숫자가 작을수록 글자의 크기가 커집니다.


    <section>

    일반적으로 문서의 컨텐츠 영역을 의미합니다.

    • 컨텐츠와 관련된 한 가지 주제 영역을 말합니다.
    • section 태그는 문장이나 스타일링 요소가 아니기 때문에 편의성이나 영역을 위함이면 div 태그가 좋습니다.
    • section 태그는 제목이 없는 경우는 섹션이라고 할 수 없기 때문에 제목을 제공해야 합니다.
    • section 태그는 일반적인 주제가 아니라면 구체적인 요소(article, aside, nav)를 사용하는 것이 더 적절합니다.
    • section 태그 안에는 또 다른 section 태그를 넣을 수 있습니다.

    <article>

    컨텐츠의 독립적인 항목을 나타내는 컨텐츠를 의미합니다.

    • article은 신문기사, 잡지, 블로그의 항목, 게시판 글 등의 독립적인 항목을 나타냅니다.
    • section 하나의 주제를 나타낸다면 article은 주제를 묶은 독립적인 콘텐츠입니다.
    • section 요소 안에는 article요소를 쓸 수 있으며, article 요소 안에도 section 요소를 쓸 수 있습니다.

    <aside>

    웹 문서의 메인 콘텐츠에 영향을 미치지 않는 콘텐츠 영역을 나타냅니다.

    • aside는 메인 컨텐츠에 영향을 미치지 않는 사이드 정보, 광고 등 부분적인 정보를 그룹화할 때 사용합니다.

    <nav>

    페이지 내에서 이동할 수 있는 네비게이션 링크 그룹입니다.

    • nav는 문서의 핵심적인 페이지의 메뉴 및 서브 메뉴에서 사용하고 문서에서 주로 한 번만 사용합니다.
    • 문서 안에 링크가 포함된 컨텐츠는 nav를 사용하지 않습니다.
    • nav는 핵십적인 네비게비션에 사용해야 하므로, 푸터 내에 링크 그룹의 사용은 적절하지 않습니다.

    <main>

    웹 문서에 주요 컨텐츠 영역을 나타낼 때 사용합니다.

    • main은 웹 페이지에서 한 번만 사용할 수 있으며, 접근성과 검색영역 노출을 향상시킵니다.
    • article, aside, footer, header, nav를 하위 요소로 사용할 수 있습니다.

    <header>

    웹 문서에 헤더 영역을 나타냅니다.

    • header에는 웹 페이지에 대한 소개, 네비게이션 영역, 테이블 영역, 검색 영역, 로고 영역을 포함하는 영역입니다.
    • header에는 제목태그가 포함될 수 있으며, 필수 조건을 아닙니다.
    • header는 섹션 콘텐츠가 아닌 그룹화하기 위한 요소이므로 section 요소를 포함할 수 없습니다.

    <footer>

    웹 문서에 푸터 영역을 나타냅니다.

    • footer는 저작권 정보, 회사 정보, 관련 링크, 주소, 바닥글, 사이트 정보 등을 포함하는 컨텐츠 영역입니다.
    • footer는 섹션 콘텐츠가 아닌 그룹을 나타내는 요소이며, section, article, aside 등을 포함할 수 있습니다.

    <HTML5에서 새로 생긴 태그>

    • article : 문서내에 독립적인 컨텐츠를 정의합니다.
    • aside : 문서내에 사이드 컨텐츠를 정의합니다.
    • audio : 오디오 파일을 정의합니다.
    • bdi : 텍스트의 출력 방향을 브라우저가 판단해 표현합니다.
    • bdo : 텍스트의 방향을 설정합니다.
    • canvas : 그림을 그리는 비트맵을 정의합니다.
    • data : 주어진 내용을 기계가 읽을 수 있도록 설정합니다.
    • datalist : 미리 정의된 옵션의 리스트를 보여줍니다.
    • details : 추가정인 정보를 보여주거나 숨기게 할 수 있습니다.
    • dialog : 추가정인 정보를 팝업(박스) 형식으로 보여줍니다.
    • figcaption : figure 요소의 제목을 정의합니다.
    • figure : 그림, 도표, 사진, 코드등의 주석으로 표시합니다.
    • main : 문서의 주요 콘텐츠를 정의합니다.
    • mark : 참고 표시를 정의합니다.
    • meter : 범위를 측정하는 게이지 정보를 정의합니다.
    • nav : 네비게이션 링크를 정의합니다.
    • wbr : 줄바꿈 할 위치를 정의합니다.

    <HTML5에서 없어진 태그>

    • applet : 자바 애플릿 삽입 / 대체 수단 : embed / object 요소 사용합니다.
    • acronym : 생략된 글자나 약어를 표시하는데 사용합니다.
    • basefont : 문서의 기본 폰트, 사이즈, 종류를 설정합니다.
    • big : 텍스트 크기를 크게 해줍니다.
    • blink : 텍스트를 깜박이게 설정합니다.
    • center : 텍스트를 가운데 정렬합니다.
    • canvas : 그림을 그리는 비트맵을 정의합니다.
    • dir : 파일 및 폴더의 디렉토리를 정의합니다.
    • frame : 프레임 / 대체 수단 : iframe과 CSS 이용하거나 서버 사이트로 컨텐츠 결합
    • frameset : 프레임 세트 / 대체 수단 : iframe과 CSS를 이용하거나 서버 사이드 컨텐츠 결합
    • noframes : 프레임에 대응하지 않는 브라우저를 위한 컨텐츠 / 대체 수단 : iframe과 CSS를 이용하거나 서버 사이드로 컨텐츠 결합
    • isindex : 서버로 송신하는 텍스트 입력란 / 대체 수단 : form과 텍스트 필드의 조합
    • big : 텍스트 사이즈를 크게 함 / 대체 수단 : CSS 사용
    • blink : 문자를 점멸 / 대체 수단 : CSS 사용
    • s : 취소선 / 대체 수단 : CSS 사용
    • u : 텍스트에 밑줄을 그음 / 대체 수단 : CSS 사용
    • bgsound : 배경음 삽입 / 대체 수단 : audio 요소를 대신 사용
    • tt : 글꼴을 같은 폭으로 표시 / 대체 수단 : CSS 사용
    • strike : 취소선 / 대체 수단 : CSS 사용

    <HTML5에서 의미가 변경된 태그>

    • a : href 없이 사용하면 "작업용 링크(null link)"로 사용 함. 문장보다는 흐름을 나타내는 콘텐츠에 사용 될 수 있다.
    • address : 섹션 개념을 통해 새로운 영역을 표시하고, 실제 우편물 주소를 표시합니다.
    • b : 인쇄상 강조 표현을 가진 텍스트 뿐만 아니라 제품 소개 내 제품명, 문서 키워드 같은 아주 특별한 중요도는 갖지 않으나 일반적인 강조의 목적에 사용합니다.
    • em : 중요한 정보를 강조할 때 사용합니다.
    • hr : 단락 단위의 주제 바꿈을 할 때 사용합니다.
    • i : 인쇄상 기울임 표현을 가진 텍스트 뿐만 아니라 소리나 분위기를 대체하는 문구 등에 사용합니다.
    • menu : 실제적 문서 메뉴 정보를 제공하는 데 사용합니다.
    • small : 세부 주석 및 법적(copyright) 인쇄 문서에서 작은 인쇄 정보를 담을 때 사용합니다.
    • strong : 강조보다 더 중요한 정보를 제공할 때 사용합니다.

    <Input>

    사용자에게 입력을 받을 수 있는 필드를 생성하는 태그입니다.

  • <hidden> : 사용자에게는 보이지 않지만 서버로 넘겨지는 값을 가집니다.
  • <text> : 한 줄짜리 텍스트를 입력할 수 있는 텍스트 상자를 넣습니다.
  • <search> : 검색 상자를 넣습니다.
  • <tel> : 전화번호 입력 필드를 넣습니다.
  • <url> : url 주소를 입력할 수 잇는 필드를 넣습니다.
  • <email> : 메일 주소를 입력할 수 있는 필드를 넣습니다.
  • <password> : 비밀번호를 입력할 수 있는 필드를 넣습니다.
  • <datetime> : 국제 표준시(UTC)로 설정된 날짜와 시간(연, 월, 일, 시, 분, 초, 분할 초)를 넣습니다.
  • <datetime-local> : 사용자가 있는 지역을 기준으로 날짜와 시간(연, 월, 일, 시, 분, 초, 분할 초)를 넣습니다.
  • <date> : 사용자 지역을 기준으로 날짜와 시간(연, 월, 일)을 넣습니다.
  • <month> : 사용자 지역을 기준으로 날짜와 시간(연, 월)을 넣습니다.
  • <week> : 사용자 지역을 기준으로 날짜와 시간(연, 주)를 넣습니다.
  • <time> : 사용자 지역을 기준으로 시간(시, 분, 초, 분할 초)를 넣습니다.
  • <number> : 숫자를 조절할 수 있는 화살표를 넣습니다.
  • <range> : 숫자를 조절할 수 있는 슬라이드 막대를 넣습니다.
  • <color> : 색상 표를 넣습니다.
  • <checkbox> : 주어진 항목에서 2개 이상 선택 가능한 체크박스를 넣습니다.
  • <radio> : 주어진 항목에서 1개만 선택할 수 있는 라디오 버튼을 넣습니다.
  • <file> : 파일을 첨부할 수 있는 버튼을 넣습니다.
  • <submit> : 서버 전송 버튼을 넣습니다.
  • <image> : 버튼 대신 사용할 이미지를 넣습니다.
  • <reset> : 리셋 버튼을 넣습니다.
  • <button> : 버튼을 넣습니다.

    • <시멘틱 태그>

    • header : 화면 상단에 위치하며 보통 로고, 로그인, 회원가입 등을 포함합니다.
    • nav : 보통
      영역의 아래쪽에 위치하며 메뉴를 표현할 수 있습니다.
    • aside : 사이드에 위치하는 공간을 의미합니다. 보통 광고, 검색 기능, 카테고리 등을 표현합니다.
    • section : 하나의 주제를 그룹화할 때 사용합니다. (ex 뉴스 기사의 제목) 여러 중심 내용을 감싸는 공간을 의미합니다.
    • article : 하나의 주제에 대해서 그 내용에 대해 설명할 떄 사용합니다. (ex 뉴스 기사, 블로그 글) 글자가 많이 들어가는 부분을 의미합니다.
    • footer : 화면 하단에 위치하며 보통 이메일, 저작권 표시 등을 표현합니다.

    <X-UA-Compatible>

    • IE=5 : 관용모드(quirks mode)로 지정된 DOCTYPE에 상관없이 IE5 렌더링 방식이 사용됩니다.
    • IE=7 : IE7 표준모드로 지정된 DOCTYPE에 상관없이 IE7 표준 모드 렌더링 방식이 사용됩니다.
    • IE=EmulateIE7 : IE7 에뮬레이션 모드로 지정된 DOCTYPE에 따라 IE7 표준모드나 관용모드로 렌더링됩니다.
    • IE=8 : IE8 표준모드로 지정된 DOCTYPE에 상관없이 IE8 표준모드로 렌더링됩니다.
    • IE=EmulateIE8 : IE8 에뮬레이션 모드로 지정된 DOCTYPE에 따라 IE8 표준모드나 관용모드로 렌더링됩니다.
    • IE=edge : 최신모드로 지정된 DOCTYPE에 상관없이 IE8 이상 버전에서 항상 최신 표준 모드로 렌더링됩니다.

    <조건부 주석>

    • ! : 아니다(not) - 예) [if !ie] ie가 아니라면
    • lt : 작다(less than) - 예) [if lt ie 9] ie9 보다 작다면
    • lte : 작거나 같다(less than equal) - 예) [if lte ie 8] ie8 보다 작거나 같다면
    • gt : 크다(greater than) - 예) [if gt ie 6] ie6 보다 크다면
    • gte : 크거나 같다(greater than equal) - 예) [if gte ie 7] ie7 보다 크거나 같다
    • () : 우선처리
    • & : 그리고(and) - 예) [if (gte ie 7)&(lt ie 9)] ie7 이상이고 ie9 미만이라면
    • | : 또는(or) - 예) [if (ie 7)|(ie 8)] ie7 이거나 ie8 이라면
